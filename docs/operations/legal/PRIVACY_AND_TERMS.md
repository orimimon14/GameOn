# 2. Privacy Policy — Draft

## 2.1 מבוא וזהות הבקר

ברוכים הבאים ל-**Swish & Game**.

Swish & Game היא אפליקציית web להתאמת גיימרים, שמאפשרת למשתמשים ליצור פרופיל, להוסיף משחקים ופלטפורמות, לגלות שחקנים אחרים, לבצע התאמות דרך discovery/swipe, לנהל chat עם matches, להשתמש ב-shop/cosmetics, לרכוש מנוי Pro, ולהשתמש בתכונות AI מסוימות.

מדיניות פרטיות זו מסבירה, בטיוטה, אילו סוגי מידע עשויים להיאסף ולעובד במסגרת השירות, כיצד נעשה בהם שימוש, עם מי הם עשויים להיות משותפים, ומהן זכויות המשתמשים.

**בקר המידע / Controller:** `[שם הישות המשפטית]`  
**כתובת:** `[כתובת]`  
**יצירת קשר בנושאי פרטיות:** `[מייל יצירת קשר]`

> הערה משפטית: יש לאשר עם עורך דין את זהות הבקר, חובות הרישום/הדיווח, נוסח ההסכמה, תחולת GDPR, חוקי הגנת פרטיות בישראל, ודינים נוספים שעשויים לחול.

## 2.2 אילו נתונים אנחנו אוספים

אנחנו שואפים לאסוף ולעבד רק מידע שנדרש להפעלת השירות, לשיפורו, לאבטחתו, ולשמירה על בטיחות המשתמשים.

הפירוט להלן מבוסס על מודל הנתונים הנוכחי של Swish & Game. אין לכלול במדיניות נתונים שאינם נאספים בפועל ללא עדכון `DATA_MODEL.md` ואישור legal.

### 2.2.1 נתוני חשבון והרשמה

בעת יצירת חשבון או התחברות, אנו עשויים לעבד:

| סוג מידע | דוגמאות | הערות |
|---|---|---|
| מזהה משתמש | `uid` | מזהה פנימי/pseudonymous. |
| פרטי התחברות | email, auth provider | מנוהל דרך Firebase Auth / Google. |
| סטטוס חשבון | `isSuspended`, `isDeleted`, `createdAt`, `updatedAt` | משמש לאבטחה, אכיפה וניהול חשבון. |
| מידע פרטי לחשבון | `users/{uid}/private/account` | גישה מוגבלת; עשוי לכלול `moderationState`. |

ייתכן ש-Firebase Auth או ספק ההתחברות יעבדו מידע נוסף בהתאם למדיניות שלהם.

### 2.2.2 נתוני פרופיל

המשתמש עשוי למסור או לעדכן פרטי פרופיל, כגון:

| סוג מידע | דוגמאות | הערות |
|---|---|---|
| פרופיל ציבורי | display name / public profile fields | לא לכלול email בפרופיל ציבורי. |
| Bio | טקסט חופשי שהמשתמש מוסר | UGC; כפוף ל-Community Guidelines. |
| תמונת פרופיל / banner | קובץ תמונה או reference לתמונה | מאוחסן דרך Firebase Storage; כפוף לכללי בטיחות. |
| cosmetics / equipped items | avatar border, banner, background | items וירטואליים. |
| Pro badge | `verifiedBadge` | משמעותו Pro member בלבד, לא אימות זהות. |

> חשוב: `verifiedBadge` אינו אימות זהות, אינו אימות גיל, ואינו הצהרה שהמשתמש “בטוח”.

### 2.2.3 נתוני משחקים, רמות ופלטפורמות

כדי להתאים בין משתמשים, אנו עשויים לעבד:

| סוג מידע | דוגמאות |
|---|---|
| משחקים | `game_id`, למשל `valorant` |
| פלטפורמות | enum values באנגלית, למשל `pc`, `playstation`, `xbox`, `mobile` |
| רמת מיומנות | `beginner`, `intermediate`, `pro`, `elite` |
| Rank / העדפות משחק | מידע שהמשתמש בוחר למסור |
| Looking for / preferences | העדפות התאמה אם קיימות במוצר |

### 2.2.4 Discovery, matches ו-chat

כדי להפעיל discovery והתאמות, אנו עשויים לעבד:

| סוג מידע | דוגמאות | הערות |
|---|---|---|
| Swipes | like/skip, target user, game | מנוהל backend-authoritative. |
| Matches | `matches/{matchId}` | נוצר לאחר התאמה הדדית. |
| Chats | `chats/{chatId}` | פתוחים למשתתפים בלבד. |
| Messages | text messages, message metadata | תוכן chat הוא UGC ורגיש. |
| Chat media | תמונות אם feature זמין ולפי gating | Pro-only לפי התכנון הנוכחי. |

אין להשתמש ב-analytics או logs לשמירת raw chat text או תוכן מדיה.

### 2.2.5 Coins, cosmetics ו-shop

ב-MVP, coins הם מטבע וירטואלי פנימי הניתן/נצבר בתוך השירות, ואינו נרכש בכסף אמיתי.

אנו עשויים לעבד:

| סוג מידע | דוגמאות | הערות |
|---|---|---|
| יתרת coins | `users/{uid}.coins` | server-owned. |
| transactions | `users/{uid}/transactions` | audit לשינויי coins. |
| owned items | cosmetics שנרכשו ב-coins | ללא ערך כספי. |
| equipped items | cosmetics פעילים בפרופיל | מוצגים לפי עיצוב המוצר. |

Coins אינם כסף אמיתי, אינם ניתנים לפדיון, ואין `cash-out`.

### 2.2.6 מנוי Pro ותשלומים

כאשר משתמש רוכש מנוי Pro, התשלום מתבצע דרך ספק תשלומים חיצוני שייקבע בהמשך (`TBD`).

אנו עשויים לעבד:

| סוג מידע | דוגמאות | הערות |
|---|---|---|
| סטטוס מנוי | `subscriptionStatus`, `subscriptionTier`, `subscriptionExpiresAt` | מנוהל backend-side. |
| פרטי entitlement | האם המשתמש Pro | נקבע רק לפי webhook/reconciliation מאומת. |
| billing metadata | provider, subscription id, status | ללא פרטי כרטיס מלאים. |
| billing events | `billingEvents` בעת Scale/V1 | normalized metadata בלבד. |

אנחנו לא אמורים לאחסן פרטי כרטיס מלאים או raw payment payload בתוך Swish & Game. פרטי תשלום יעובדו על ידי ספק התשלומים החיצוני ובהתאם למדיניות שלו.

### 2.2.7 שימוש, analytics ונתוני מכשיר

אנו עשויים למדוד שימוש במוצר כדי להבין בריאות מוצרית ותפעולית, בהתאם ל-`ANALYTICS.md` ו-`OBSERVABILITY.md`.

דוגמאות:

| סוג מידע | דוגמאות |
|---|---|
| אירועי מוצר | `signup_completed`, `discovery_opened`, `match_created` |
| שימוש במערכת | latency, errors, function failures |
| context טכני | `app_env`, `app_version`, device type, route |
| pseudonymous identifiers | `uid` או מזהה אנליטיקה pseudonymous אם נדרש |

אין לשלוח ל-analytics:

- email.
- bio.
- raw chat.
- payment data.
- Gemini prompts/responses.
- secrets.
- report descriptions.

### 2.2.8 AI inputs ו-AI metadata

Swish & Game עשויה להציע תכונות AI, כגון profile review או squad advice, באמצעות Google Gemini בצד השרת בלבד.

כאשר משתמש מפעיל תכונת AI, אנו עשויים לעבד:

| סוג מידע | דוגמאות | הערות |
|---|---|---|
| input שנמסר לתכונת AI | בקשה/קלט שהמשתמש מספק לצורך הפקת תשובה | מעובד כדי לספק את השירות. |
| פרטי פרופיל רלוונטיים | למשל games/preferences הדרושים לפונקציה | רק לפי צורך. |
| AI request metadata | `aiRequests/{requestId}` | audit/status/cost metadata. |
| סטטוס בקשה | `pending`, `completed`, `failed`, `blocked` | לצורכי תפעול ובטיחות. |

בהתאם לתכנון הנוכחי, אין לשמור ב-logs או analytics raw Gemini prompts/responses, system prompts, או API keys. יש לאשר משפטית וטכנית את מדיניות retention עבור inputs/outputs של AI לפני launch.

### 2.2.9 דיווחים, חסימות ובטיחות

לצורכי Trust & Safety, אנו עשויים לעבד:

| סוג מידע | דוגמאות |
|---|---|
| reports | `reports/{reportId}` |
| report reason | `harassment`, `hate_speech`, `sexual_content`, `scam_spam`, `underage_concern`, `cheating_exploits`, `fake_profile`, `other` |
| blocks | `users/{uid}/blocks/{blockedUid}` |
| moderation state | `users/{uid}/private/account.moderationState` |

Report descriptions ומידע מודרציה הם רגישים ואינם נגישים למשתמשים רגילים.

## 2.8 זכויות משתמש

בכפוף לדין החל, משתמשים עשויים להיות זכאים ל:

- גישה למידע אישי.
- תיקון מידע.
- מחיקת חשבון או מידע.
- ייצוא מידע.
- התנגדות או הגבלה לעיבוד מסוים.
- ביטול הסכמה כאשר העיבוד מבוסס על הסכמה.
